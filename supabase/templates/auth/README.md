# Supabase Auth email templates

HTML templates that match the MomentoVino brand (WINE `#722F37`, BG `#F5EBE0`,
serif headlines, sans body). They use Go template variables (`{{ .Var }}`)
that Supabase exposes in each email type.

Only the templates currently used by the app live here. The other Supabase
auth emails (Magic link, Invite user, Change email, Reauthentication) are
**not** wired up — sign-in is via Apple, Google, or email + password only.

## Install

Supabase Dashboard → **Authentication → Email Templates**:

| Dashboard label | File                    | When it fires                                  |
|-----------------|-------------------------|------------------------------------------------|
| Confirm signup  | `confirmation.html`     | `supabase.auth.signUp()` with email confirmation on |
| Reset password  | `reset-password.html`   | `supabase.auth.resetPasswordForEmail()`        |

> The "Reset password" template is internally called `recovery` in Supabase's
> templating engine — that's why this folder previously had `recovery.html`.
> The file is now named after the user-facing action.

Paste each file's HTML into the matching template editor and save.

## Subjects (set in the same screen)

- Confirm signup: `Welcome to MomentoVino — confirm your email`
- Reset password: `Reset your MomentoVino password`

## Notes on rendering

- Email clients strip `<link>` to Google Fonts; we declare brand fonts AND
  fall back to system serifs/sans. Most clients (Gmail, Apple Mail) render
  the system fallback — that's fine.
- Layout is single-column, max-width 560px, fully fluid below that.
- The CTA button is a table-based row with inline padding so Outlook honors it.
- Plain-text alt is auto-generated by Supabase; no need to provide one.

## Design system reference

Every new template **must** follow the MomentoVino brand palette. Email clients
don't support CSS vars or external stylesheets, so paste the hex codes inline.

| Token  | Hex       | Use                                                |
|--------|-----------|----------------------------------------------------|
| WINE   | `#722F37` | Headlines, primary CTA background, brand wordmark  |
| INK    | `#3F2A2E` | Body copy                                          |
| SUBTLE | `#6E5A5E` | Secondary copy, footer, fine print                 |
| BG     | `#F5EBE0` | Email body background                              |
| BORDER | `#E8DDD4` | Card border, dividers                              |
| Card   | `#FFFFFF` | Inner content card                                 |

The same palette is declared at the top of every mobile screen — see
[apps/mobile/app/onboarding/save-account.tsx](../../../apps/mobile/app/onboarding/save-account.tsx)
for an example. Formal tokens live in
[packages/design-tokens/tokens.json](../../../packages/design-tokens/tokens.json)
and [packages/design-tokens/mobile.ts](../../../packages/design-tokens/mobile.ts)
(`colors.primary[500]` = WINE).

Fonts: `'DM Serif Display','Playfair Display',Georgia,serif` for headlines,
`'DM Sans','Helvetica Neue',Arial,sans-serif` for body.

When creating a new template, copy [reset-password.html](reset-password.html)
as the starting point — it has the canonical layout (560px max-width table,
brand wordmark, white card, table-based CTA with `border-radius:50px`,
mobile breakpoint at 600px).
